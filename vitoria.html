Por que utilizamos metodologias ágeis ao invés da tradicional/cascata?

A metodologia de gerenciamento de projetos tradicional (também chamada de cascata), prega que uma etapa só pode ser iniciada se a outra estiver 100% completa. Exemplo: - Os testes de um software só iniciarão quando toda a codificação estiver finalizada; Já as metodologias ágeis foram criadas para lidar com projetos que requerem mudanças constantes e rápidas, incertezas e muitas variáveis. - Nesse modelo de gestão, muitas vezes não é necessário terminar uma etapa para iniciar a outra, e muito menos realizar as etapas para o projeto todo. A ideia é dividir o projeto em pequenos pedaços e aplicar as etapas do desenvolvimento de software nesses pedaços pequenos.

Qual a importância de fazermos uma boa descrição de merge request?

É importante descrever o que foi feito no arquivo para saber do que se trata ou quem for realizar os testes saber exatamente o que fazer.

Fale brevemente sobre as etapas de desenvolvimento de software:

Análise

A análise é ver a necessidade do cliente/empresa, como funciona o dia a dia, qual a necessidade, recolher diversas informações para serem gerenciadas, pois vão demandar o auxílio do software.

Desenvolvimento

O desenvolvimento de software é composto por diversas etapas, e que mesclam diversos setores da empresa que foi analizada. Tudo começa com a necessidade de um software para solucionar algum problema de negócio, até a entrega de uma primeira versão.

Teste

A fase de tese é para verificar se o software realmente atende a necessidade do cliente ou se tem que ser alterado mais alguma coisa para concluir.

Concluído

Após a verificação do teste o software é entrege ao cliente.
